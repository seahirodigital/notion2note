# 【仕様書】Notion記事のNote自動投稿システム（Chrome拡張機能 ＆ GitHub Actions）

Notionに保存されている記事を、Chrome拡張機能を「リモートスイッチ」として、GitHub Actions経由でNoteブログへ自動アップロード・投稿するシステム。



## 0. 使い方のゴール
・Notionの記事ページを見ている状態で、Chrome拡張内のボタンを推すだけでNoteへの記事投稿が完了する。
・記事投稿完了とは
ー記事添付
ーアフィリエイトリンクのOGPを展開
ー画像の添付
ー目次の挿入
ー不要な文字起こしの削除された状態
ー投稿前のタグ設置
ーサムネイル画像の添付

ただしこれらの機能はすでに、利用にあたり必読のリンク先野中のソースコードですでに実現しているものばかりであり、新規作成が必要なコードは少ないため、お前が０から作ろうとするな、まずは類似機能を探スことから始めろ。
---

## 1. 全体構造・システムアーキテクチャ

本システムは、Chrome拡張機能とGitHub Actionsの2つのフォルダに完全に分離して開発を行います。

* **Chrome拡張機能（スイッチ役）**
* ユーザーがブラウザ上でボタンをクリックすると、GitHub API（Repository Dispatch等）を叩き、GitHub Actionsをリモートで起動する。


* **GitHub Actions（実行・パース役）**
* 既存のNote投稿スクリプトをベースに、Notionからのデータ取得、記事の整形、Noteへの自動投稿を実行する。



---

## 2. Chrome拡張機能 仕様

* **フォルダ構成:** GitHub Actions用とは別に専用フォルダに独立させる。
* **メイン機能:** ボタン押下により、特定のGitHubワークフローをトリガーする。
* **認証・設定:** * ブログのIDやパスワード、各種トークン類は、既存環境とは別に管理。
* `.env` ファイルなどの外部設定ファイルで容易に変更できるようにする。
* ※セッションログイン等の既存機能はそのまま流用し、ソースコードを無駄に変更しない。



---

## 3. GitHub Actions / 投稿スクリプト 仕様（新規実装・変更点）

すでに動いている既存コードのロジックをベースとし、「すでに動いている部分は下手に変更しない」ことを大前提に、以下の追加・変更を行います。

### 3.1. Notionブロックからの画像取得（新規実装ポイント）

* **概要:** Notionはブロック形式でデータが保持されているため、画像ブロックを適切に解析・取得するロジックを実装する。
* **配置ルール:** 取得した画像はすべて、記事内の「エグゼクティブサマリー」というH2の段落テキストの直後（一番最後）に配置する。
* **サムネイル:** Notionの一番上部にある画像をサムネイル画像として、設定する（つまり、ブログ記事の画像となる）

### 3.2. アフィリエイトリンクのランダム挿入（ロジック変更）

* **概要:** 用意されたアフィリエイトリンクのテキスト（例: 全20個）から、ランダムに複数個（例: 2個）を選出して記事内に挿入する。
* **挿入位置:** * 各H2の終わり（次の段落が始まる前）に挿入する。
* （例: 最初のH2の終わりに2個挿入したら、次のH2の終わりにも同様にランダムに挿入していく）。



### 3.3. 不要テキストの除外

* **ルール:** 記事内にある「文字起こし」というH2以降の生の文章（テキスト）はすべて不要とし、Noteへの投稿内容からは除外する。

### 3.4. YouTube URLの配置

* **ルール:** Notion内のヘッダーデータベースに含まれるYouTubeのURLは、Note記事の最上位（文頭）に配置する。

### 3.5. 外部ファイルによるタグ管理

* **概要:** 投稿時に設定するタグを外部ファイルで管理できるようにする。
* **実装方法:** `tag.md` というファイルを新規作成し、その中に記述されたタグを投稿前に読み込んで貼り付ける。
* **注意点:** すでにタグを貼り付けるコード自体は存在するため、**貼り付ける「内容」を外部ファイル化するのみ**とし、既存の貼り付けコード自体は変更しない。

### 3.6. 目次の追加位置

* **ルール:** 目次は、記事内の**最初のH2の前**に追加する。

### 3.7. 既存機能の維持（変更禁止）

* **OGP展開:** Noteの投稿編集画面でOGP（リンクプレビュー）を展開する既存のコードは、一切変更せずそのまま流用する。

---

## 4. 開発にあたっての共通遵守事項

* **既存コードの尊重:** すでに正常に動作している実績があるため、指定された変更箇所以外のあらゆるファイル、ソースコード、ロジックへ下手に手を加えないこと。
* **環境の分離:** ブログIDやパスワード等の認証情報は環境変数（`.env` や GitHub Secrets）で管理し、ポータビリティを担保すること。
